# AI_USAGE

## 사용한 AI 도구

- ChatGPT / Codex

## 활용 방식

AI는 과제 요구사항을 해석하고, 기존 코드 흐름을 이해하는 데 사용했습니다. 특히 서버 실패 처리, 낙관적 업데이트, rollback, race condition, virtualization, 검색/필터 구조처럼 제가 헷갈리는 부분을 질문하고 설명을 들었습니다.

코드를 모두 그대로 맡긴 것은 아니고, 구현 방향을 먼저 이해한 뒤 직접 수정하면서 확인했습니다. AI가 제안한 내용도 바로 적용하지 않고, 현재 코드 구조와 과제 요구사항에 맞는지 계속 비교했습니다.

## 그대로 사용한 부분

다음 부분은 AI가 제안한 방향을 대부분 받아들여 구현했습니다.

- 실패한 쓰기 요청에서 `catch`로 오류를 처리하고 rollback하는 구조
- 이동/수정/삭제/생성에서 낙관적 업데이트 후 실패 시 이전 상태로 되돌리는 방식
- 빠른 연속 이동에서 오래된 응답을 무시하기 위한 요청 순번 비교 방식
- 5,000개 렌더링 부담을 줄이기 위한 `@tanstack/react-virtual` 도입
- 검색과 우선순위 필터를 순수 함수로 분리하고 테스트를 추가하는 방향
- 409 충돌과 500 서버 오류를 toast 메시지로 구분하는 방향

## 검증하거나 수정한 부분

AI가 제안한 내용을 그대로 쓰지 않고, 직접 확인하거나 수정한 부분도 있었습니다.

- 처음에는 실패 처리를 단순히 `catch`만 추가하면 된다고 생각했지만, 화면을 먼저 바꾸는 낙관적 업데이트에서는 실패 시 이전 상태로 되돌리는 rollback이 함께 필요하다는 점을 확인했습니다.
- 카드 이동에는 서버의 `version`만으로 충분할 것 같다고 생각했지만, `version`은 서버 충돌 확인용이고 빠른 연속 이동의 응답 순서 문제는 별도의 요청 순번으로 처리해야 한다는 점을 확인했습니다.
- 가상화 적용 후 카드 높이가 바뀌는 인라인 수정 폼에서 카드가 겹치는 문제가 있었습니다. 단순히 `estimateSize` 값을 조정하는 것으로 끝내지 않고, `measureElement`와 `data-index`를 사용해 실제 row 높이를 다시 측정하도록 수정했습니다.
- 생성 UI는 모달로 만들 수도 있었지만, 과제 규모와 현재 화면 흐름을 고려해 보드 상단의 인라인 폼으로 구현했습니다.
- 실패 알림은 `alert`도 고려했지만, 사용자의 작업 흐름을 막지 않는 toast가 더 적절하다고 판단했습니다.
- AI가 문서 초안을 너무 넓게 작성했을 때, 과제에서 요구한 DECISIONS 항목에 맞지 않는다고 판단해 각 질문에 맞춰 다시 정리했습니다.
- 대량 데이터 성능 설명에서 정밀한 Performance profiling을 한 것처럼 쓰는 것은 맞지 않다고 판단해, 실제로 확인한 수동 테스트와 DOM 개수 확인 수준으로 표현을 수정했습니다.

## 거부한 부분

다음 제안은 이번 과제 범위에 비해 과하다고 판단해 적용하지 않았습니다.

- TanStack Query 도입: 서버 상태 관리에는 장점이 있지만, 이번 과제에서는 낙관적 업데이트, rollback, race condition을 직접 구현하고 설명하는 것이 더 중요하다고 판단했습니다.
- 409 충돌 해결을 위한 비교/병합 UI: 좋은 방향이지만 구현 범위가 커져서 이번에는 409와 500을 구분해 안내하고 rollback하는 수준으로 제한했습니다.
- 실패한 쓰기 요청의 자동 재시도: 자동 재시도는 편리하지만, 쓰기 요청에서는 순서 꼬임이나 중복 처리 문제가 생길 수 있어 수동 재시도를 선택했습니다.
- 다중 탭 동기화, 키보드 카드 이동, 검색 debounce, 상태/태그 다중 필터는 시간이 부족해 우선순위에서 제외했습니다.

- 브라우저에서 5,000개 데이터 기준 스크롤, 드래그, 생성, 수정, 삭제, 검색, 우선순위 필터 동작 확인
- 실패율을 조정해 저장 실패 시 rollback과 toast 메시지 확인
- 빠른 연속 이동 시 마지막 이동 상태가 유지되는지 확인
- 콘솔에서 `document.querySelectorAll('.card').length`로 실제 DOM 카드 수가 전체 5,000개보다 적게 유지되는지 확인
